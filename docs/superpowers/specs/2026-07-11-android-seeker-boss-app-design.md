# 微招 Android 双角色 App 设计说明

**日期：** 2026-07-11  
**范围：** 求职者端、招聘者（Boss）端；管理员端明确排除，继续使用 Web 管理后台。

## 1. 产品边界

Android 使用同一个安装包。登录成功后根据服务端返回的 `role` 进入求职者或招聘者导航。App 不提供管理员登录入口，也不调用 `/api/admin/**`。投诉和申诉由两类用户在 App 提交，审核仍在 Web 管理后台完成。

## 2. 视觉适配

继承 Web 端 `app-shell.css` 的品牌设计，而不是机械缩放网页：

- 品牌主色：`#0F766E`，强调色：`#00A896`，浅青背景：`#ECFEFF`。
- 页面背景：珠光白 `#F5F7FB`；卡片：纯白 `#FFFFFF`；边框：`#E2E8F0`。
- 主标题：`#0F172A`；正文：`#334155`；辅助文字：`#64748B`。除按钮反白文字外，不使用白色正文。
- 卡片圆角 14dp，按钮圆角 10dp，页面水平边距 16dp，触控区域至少 48dp。
- 手机使用单列卡片、顶部标题栏和底部导航；详情、编辑和会话使用独立页面。
- 文件和图片上传使用系统文件选择器，并提供缩略图、删除和上传进度。

## 3. 信息架构

### 求职者

底部导航：职位、投递、消息、论坛、我的。

- 职位：搜索、城市/学历/经验/薪资筛选、详情、公司、收藏、投递、投诉。
- 投递：投递状态、撤回、面试信息、联系 Boss。
- 消息：通知、聊天列表、聊天详情、附件。
- 论坛：板块、搜索、图文帖子、标签、喜欢、评论、收藏。
- 我的：简历、头像、附件、收藏、投诉记录、申诉、退出。

### 招聘者

底部导航：工作台、职位、人才、消息、我的。

- 工作台：公司审核状态、职位和投递统计、待处理事项。
- 职位：发布、编辑、关闭、重新开放、职位统计。
- 人才：投递列表、简历详情、状态流转、面试安排、投诉。
- 消息：通知、会话和附件。
- 我的：公司资料、论坛、投诉记录、申诉、退出。

## 4. 移动端接口

- 新增 `/api/mobile/auth/login|me|logout`，登录请求为 JSON，验证码仍复用 Redis 两分钟有效期和刷新防刷策略。
- 新增 `/api/mobile/boss/**` JSON 接口，复用现有 Boss Service，不从 HTML 中解析数据。
- 求职者读接口复用 `/api/seeker/**`，补齐投递、收藏、撤回和上传等 JSON 写接口。
- 论坛、聊天、投诉、申诉继续复用现有 `/api/forum/**`、`/api/chat/**`、`/api/complaints`、`/api/appeals`。
- App 使用 JWT Bearer；验证码请求和登录同时保留 CookieJar，确保同一 `JSESSIONID`。
- 写请求携带 `Idempotency-Key`；接口按角色校验资源归属，禁止水平越权。

## 5. 环境

- `prod`：`http://superorange.top/`，仅该域名允许明文 HTTP。
- `local`：Android 模拟器访问 `https://10.0.2.2:8443/`；调试证书仅 debug 环境信任。
- URL、协议和证书策略由 product flavor 区分，不在业务代码中硬编码。

## 6. 验收标准

1. 同一 APK 能使用求职者和 Boss 账号登录，并进入各自导航；管理员账号被明确拒绝。
2. 验证码显示 2:00 倒计时，冷却期间禁用刷新并显示剩余秒数。
3. 核心列表、详情、写操作均通过真实 JSON API，不使用 WebView。
4. 所有标题和正文在珠光白背景上清晰可见，页面尺寸、导航图标和字体层级统一。
5. production flavor 可在 `http://superorange.top` 工作，local flavor 可在本机 HTTPS 环境工作。
6. 后端测试、Android 单元测试和 APK 构建通过后才允许合并主分支与部署。
