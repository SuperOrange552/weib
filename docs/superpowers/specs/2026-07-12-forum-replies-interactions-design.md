# 论坛两级评论与个人互动中心设计

## 1. 目标

完善微招论坛的互动闭环：用户可以对一级评论或已有追评进行回复；帖子详情清晰显示喜欢、收藏按钮和当前状态；登录用户可在“我的互动”页面查看并管理自己喜欢、收藏的帖子。功能同时支持求职者和招聘者身份。

## 2. 范围

本次包含：

- 两级评论与“回复 @用户”展示。
- 回复自己或他人的评论。
- 帖子喜欢、取消喜欢、收藏、取消收藏的可视化按钮。
- 帖子详情返回当前用户的喜欢/收藏状态。
- `/forum/me` 我的互动页面，包含“我的喜欢”和“我的收藏”两个标签页。
- 分页查询个人互动记录。
- 数据库增量迁移、自动化测试、生产部署与验证。

本次不包含：

- 无限层级评论树。
- 评论点赞。
- 私信提醒或推送通知。
- 管理后台评论审核页面扩展。

## 3. 评论数据模型

在 `forum_comments` 增加两个可空字段：

- `parent_id BIGINT NULL`：追评所属的一级评论 ID。
- `reply_to_comment_id BIGINT NULL`：本次回复直接针对的评论 ID。

规则：

1. 一级评论的 `parent_id` 和 `reply_to_comment_id` 均为空。
2. 回复一级评论时，`parent_id` 与 `reply_to_comment_id` 都指向该一级评论。
3. 回复追评时，`parent_id` 指向所属一级评论，`reply_to_comment_id` 指向被回复的追评。
4. 被回复评论必须存在、状态为 `ACTIVE`，并且与目标帖子相同。
5. `parent_id` 指向的记录必须是一级评论，禁止形成循环引用。
6. 允许用户回复自己的评论。
7. 帖子的 `comment_count` 包含一级评论和所有追评。

增量 SQL 使用 `ADD COLUMN`、索引和外键，执行前由生产发布脚本备份数据库。旧评论两个字段为空，自动按一级评论兼容，不需要批量改写。

## 4. 评论接口与响应

### 4.1 发布评论或追评

保留接口：

```http
POST /api/forum/posts/{postId}/comments
```

请求：

```json
{
  "content": "具体回复内容",
  "replyToCommentId": 123
}
```

- `replyToCommentId` 为空：发布一级评论。
- `replyToCommentId` 有值：服务端解析目标评论及所属一级评论并创建追评。
- 旧客户端只传 `content` 仍然有效。

### 4.2 查询评论

保留接口：

```http
GET /api/forum/posts/{postId}/comments
```

仅返回一级评论数组，每条一级评论包含 `replies` 数组。追评按创建时间升序平铺，不继续嵌套。

评论响应至少包含：

```json
{
  "id": 10,
  "postId": 5,
  "parentId": null,
  "replyToCommentId": null,
  "authorId": 6,
  "authorRole": "SEEKER",
  "authorName": "莉莉",
  "avatar": "/uploads/avatar.png",
  "replyToAuthorName": null,
  "content": "一级评论",
  "createdAt": "2026-07-12T20:00:00",
  "replies": []
}
```

## 5. 喜欢与收藏状态

现有喜欢、取消喜欢、收藏和取消收藏接口继续使用：

```http
POST   /api/forum/posts/{id}/like
DELETE /api/forum/posts/{id}/like
POST   /api/forum/posts/{id}/favorite
DELETE /api/forum/posts/{id}/favorite
```

帖子详情响应增加：

- `likedByCurrentUser`：当前登录用户是否喜欢。
- `favoritedByCurrentUser`：当前登录用户是否收藏。

匿名读取详情时两个字段均为 `false`。按钮行为：

- 未喜欢显示 `♡ 喜欢 {数量}`，点击后变为高亮的 `♥ 已喜欢 {数量}`。
- 未收藏显示 `☆ 收藏 {数量}`，点击后变为高亮的 `★ 已收藏 {数量}`。
- 再次点击执行 DELETE 并立即更新按钮状态与数量。
- 匿名点击时跳转登录页或显示明确登录提示。
- 重复请求由数据库唯一约束和现有幂等逻辑保证不会重复计数。

## 6. 我的互动接口

新增登录态接口：

```http
GET /api/forum/me/likes?page=0&size=20
GET /api/forum/me/favorites?page=0&size=20
```

返回 `PageResponse<ForumPostResponse>`，按互动时间倒序，而不是按帖子发布时间倒序。只返回状态为 `ACTIVE` 的帖子。

查询必须在数据库完成分页，不先加载全部互动记录再在内存分页。帖子卡片继续返回作者、头像、发布时间、标签、喜欢数、评论数、收藏数以及当前用户互动状态。

## 7. 页面设计

### 7.1 论坛导航

论坛首页、帖子详情和发布页的统一导航增加：

- 论坛首页。
- 我的互动。
- 发布帖子。
- 返回职位大厅或返回 Boss 工作台。

`/forum/me` 需要登录；匿名点击由登录拦截器重定向 `/login`。

### 7.2 帖子详情

互动按钮放在正文和标签下方，使用有边框、有图标、有文字的按钮，不能只显示不可点击的统计文字。按钮提供默认、悬停、已激活和禁用状态。

评论区：

- 一级评论使用独立卡片。
- 每条评论和追评右侧提供“回复”按钮。
- 点击后在对应一级评论下打开内联输入框，并显示“回复 @用户名”。
- 追评区域缩进显示，使用左侧边线区分层级。
- 提交成功后刷新当前评论树并清空输入框。

### 7.3 我的互动

新增 `/forum/me` 页面：

- 顶部两个统计卡片：喜欢帖子总数、收藏帖子总数。
- 两个可切换标签页：“我的喜欢”“我的收藏”。
- 每个帖子使用与论坛首页一致的卡片，显示标题、摘要、作者、时间、标签和三项互动数。
- 卡片提供“取消喜欢”或“取消收藏”按钮；操作成功后从当前列表移除并更新统计数。
- 空状态明确显示“还没有喜欢的帖子”或“还没有收藏的帖子”。
- 支持上一页、下一页和总数展示。

## 8. 身份与权限

- 评论、追评、喜欢和收藏都使用 `ActiveIdentityResolver` 的当前会话用户。
- SEEKER 与 BOSS 都可以使用互动功能。
- 评论作者展示使用当前 `authorRole` 对应的 `RoleProfile`，保持双身份头像和昵称隔离。
- 喜欢和收藏仍按认证用户 ID 保存；同一账号切换 SEEKER/BOSS 后共享“我的互动”记录。
- 被禁言用户不能评论或追评；被禁止发布的用户不能发布帖子，沿用现有制裁规则。

## 9. 错误处理

- 回复目标不存在、已隐藏或属于其他帖子：HTTP 业务错误，页面显示“回复的评论已不存在，请刷新后重试”。
- 未登录写操作：返回登录提示，不伪装成网络错误。
- 重复喜欢/收藏：不重复增加计数。
- 取消不存在的喜欢/收藏：保持幂等，不产生负数。
- 页面请求失败时保留用户已输入的评论内容，并允许重试。

## 10. 测试与生产验证

自动化测试覆盖：

1. 一级评论创建。
2. 回复一级评论。
3. 回复追评时正确归属同一一级评论。
4. 拒绝跨帖子回复和不存在的评论。
5. 评论树按一级评论和追评正确组装。
6. 喜欢/收藏状态随 POST/DELETE 切换且计数正确。
7. 我的喜欢/收藏按互动时间分页。
8. 论坛三类页面和“我的互动”页面存在明确按钮、状态和导航入口。
9. 旧客户端不传 `replyToCommentId` 仍可发布一级评论。

生产验证使用测试账号完成：

- 发布一级评论、回复自己的评论、回复他人的追评。
- 喜欢、取消喜欢、收藏、取消收藏。
- 在“我的互动”中看到对应帖子和数量变化。
- SEEKER/BOSS 两种活动身份页面正常。
- 检查数据库关系、Nginx/Spring 日志、Redis、systemd 健康状态和发布版本哈希。

## 11. 回滚与兼容

- JAR 可通过 `/opt/weib/scripts/rollback.sh` 回滚。
- 新增评论字段允许为空，旧版本忽略这些字段，因此数据库迁移向后兼容。
- 回滚 JAR 时无需立即删除新字段；如需反向迁移，必须在确认没有追评数据后单独执行。
